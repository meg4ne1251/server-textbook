# サーバー/Linuxシステム基礎教科書

「サーバーは触れるが、次の体系的な一冊が欲しい」という読者のための、
サーバー/Linuxシステムの個人学習用教科書です。

[`network-guide`](https://github.com/meg4ne1251/network-textbook) (ネットワーク技術教科書) と対をなす、完全に独立した分野の
プロジェクトです。プロセス・スレッド、ファイルシステム、仮想化、systemd、
高可用性、可観測性といった、サーバーOSの内部動作を扱います。

- なぜコンテナはnamespacesとcgroupsの組み合わせだけで実現できるのか
- ジャーナリングファイルシステムは、なぜクラッシュ後も一貫性を保てるのか
- systemdのUnit依存関係は、実際にはどう解決されているのか

こうした問いに、一次情報源(カーネルドキュメント・POSIX仕様・公式仕様書)を
引きながら答えていく構成です。

## 対象読者

- Linuxサーバーの操作には慣れているが、OSやカーネルが「なぜそう動くのか」を
  仕組みのレベルで説明できるほどではない人
- 実務や趣味(ホームラボ含む)で断片的な知識・勘は身についているが、
  カーネルドキュメントや仕様書レベルで動作原理を語れるほどではない人
- エンタープライズ〜大規模インフラの設計・運用を目指している人

## 構成(計画)

全9部・34章構成を予定しています。前提知識ゼロで読める第0部(入門)から立ち上げ、
各分野は「そもそも何のためにある仕組みか」という基礎から始めて、理論(カーネル
ドキュメント・仕様書ベース)→ 内部動作の詳細 → トラブルシューティングの順に
深め、最終的にサーバーの基礎・運用・応用レベルの体系的理解を目指します。

現時点では章立ての設計フェーズであり、本文の執筆はこれから着手します。

| 部 | 内容 |
|---|---|
| 第0部 サーバー・Linux入門 | サーバー/OS/カーネルの全体像、シェルとコマンドの仕組み、ファイル階層(FHS)とパーミッションの基礎、パッケージ管理 |
| 第1部 プロセス・カーネル基礎 | プロセス/スレッド、システムコール、仮想メモリ、スケジューラ、シグナル |
| 第2部 ファイルシステム・ストレージ | VFS、ジャーナリング、inode、LVM/RAID、ネットワークストレージ |
| 第3部 Linuxネットワークスタック内部 | ソケットAPI、netfilter/nftables、ネットワークネームスペース、tc/qdisc(`network-guide`との橋渡し) |
| 第4部 仮想化・コンテナ基盤 | cgroups/namespaces、KVM/QEMU、コンテナランタイム |
| 第5部 systemd・サービス管理 | Unit依存関係解決、cgroup連携、journald |
| 第6部 運用・セキュリティ | ユーザー/権限/認証の運用、SSH、パッケージ/更新管理とバックアップ、ファイアウォールと基本的な堅牢化 |

## 読み進め方

各章はおおむね次の流れで構成される予定です。

1. **概要 / 導入** — そもそも何のためにある仕組みか
2. **理論** — カーネルドキュメント・POSIX仕様ベースの動作原理(なぜこう設計されているか)
3. **内部動作の詳細** — データ構造、状態遷移、アルゴリズムなど
4. **(補助)実行例** — 理解を助ける場合のみ、最小限で
5. **トラブルシューティング** — strace/ftrace/procファイル等での見え方など
6. **演習・まとめ**

第3部(Linuxネットワークスタック内部)は [`network-guide`](https://github.com/meg4ne1251/network-textbook) 側の章
(特にVXLAN/EVPN・Linuxブリッジの実行例)と相互参照できる橋渡し章として書く予定です。
